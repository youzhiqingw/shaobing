## 🎯 项目概述

### 核心定位
- **目标**：烧饼小摊线上展示窗口
- **功能**：展示烧饼图片、收款二维码、今日出摊信息
- **特点**：零成本托管（Cloudflare 免费套餐）、响应式设计（PC/手机自适应）
- **无支付集成**：仅展示微信/支付宝收款码，顾客线下扫码购买

### 功能权限总览

| 功能 | 权限 | 说明 |
| ---- | ---- | ---- |
| **首页展示** | 公开 | 烧饼图片、今日出摊信息、收款二维码 |
| **顾客评价** | 公开 | 点赞"好吃"或"不好吃"，填写建议（可选） |
| **历史记录** | 管理员 | 查看/编辑过去每天的出摊记录 |
| **评价管理** | 管理员 | 查看所有评价详情和建议 |
| **今日状态编辑** | 管理员 | 编辑今日出摊信息 |

### 管理员操作指南

| 功能 | 操作方式 |
| ---- | -------- |
| **今日状态编辑** | 点击"✏️ 编辑" → 输入密码登录 → 修改内容 → 保存 |
| **历史记录查看** | 点击"� 历史" → 输入密码登录 → 查看列表 |
| **历史记录修改** | 在历史列表中点击某条记录 → 编辑 → 保存 |
| **顾客评价** | 点击"好吃/不好吃" → 填写建议（可选）→ 提交 |
| **评价管理** | 点击评价区"管理" → 输入密码登录 → 查看所有评价详情 |

### 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 前端 | HTML5 + CSS3 + Vanilla JS | 零构建工具，纯静态 |
| 后端 | Cloudflare Workers | 边缘计算 API |
| 存储 | Cloudflare KV | 状态/历史/评价数据存储 |
| 托管 | Cloudflare Pages | 静态网站托管 |
| 安全 | 密码验证 + 速率限制 | 管理员保护 |

## 🔧 功能模块

### 模块 1：首页展示系统（公开）

| 功能 | 说明 | 数据来源 |
|------|------|---------|
| 烧饼图片展示 | 轮播或网格展示烧饼图片 | 硬编码或 R2（可选） |
| 今日出摊信息 | 出摊状态、位置、时间、备注 | KV: `status:YYYY-MM-DD` |
| 收款二维码 | 微信/支付宝收款码图片 | 静态图片 |
| 顾客评价入口 | 好吃/不好吃按钮 | KV: `reviews` |

**今日状态数据结构：**
```javascript
// KV key: status:2026-03-18
{
  date: "2026-03-18",
  isOpen: true,              // 是否出摊
  status: "出摊中",           // 出摊中 / 已收摊 / 休息
  location: "xx 小区门口",     // 出摊位置
  time: "07:00-09:00",       // 出摊时间
  note: "今天多带了韭菜盒子",  // 备注信息
  updatedAt: "2026-03-18T07:30:00Z"
}
```

---

### 模块 2：顾客评价系统（公开）

| 功能 | 说明 | 限制 |
|------|------|------|
| 点赞评价 | 好吃/不好吃二选一 | 每 IP 每小时限投 1 次 |
| 建议留言 | 可选填写，最多 100 字 | 敏感词过滤 |
| 实时统计 | 显示好吃/不好吃人数和百分比 | 前端计算 |
| 精选展示 | 随机展示 10 条评价 | KV 缓存 |

**评价数据结构：**
```javascript
// KV key: reviews
[
  {
    id: "1710756000000",
    type: "good",              // good | bad
    content: "酥脆掉渣，太好吃了！",
    ip: "192.168.1.1",         // 脱敏处理
    createdAt: "2026-03-18T08:30:00Z"
  },
  // ... 最多保留 500 条
]
```

---

### 模块 3：历史记录系统（管理员）

| 功能 | 说明 | 操作 |
|------|------|------|
| 历史列表 | 查看最近 90 天的出摊记录 | 按日期倒序排列 |
| 历史记录 | 点击某天查看详细状态 | 只读模式 |
| 编辑历史 | 修改历史记录的出摊信息 | 编辑模式 |
| 自动归档 | 每日 23:59 自动归档今日状态 | Worker 定时触发 |

**历史记录数据结构：**
```javascript
// KV key: history
[
  {
    date: "2026-03-17",
    isOpen: true,
    status: "已收摊",
    location: "xx 学校南门",
    time: "07:00-09:00",
    note: "今天带了 50 个烧饼，全部售罄",
    updatedAt: "2026-03-17T09:00:00Z"
  },
  // ... 最多保留 90 条
]
```

### 模块 4：管理员后台（密码保护）

| 功能 | 操作 | 说明 |
|------|------|------|
| 密码验证 | 输入管理员密码 | 密码存储在 KV，比对验证 |
| 今日状态编辑 | 修改出摊状态、位置、时间、备注 | 保存到 `status:YYYY-MM-DD` |
| 历史记录管理 | 查看/编辑任意历史日期记录 | 支持搜索日期 |
| 评价管理 | 查看所有评价详情、IP、建议 | 可删除恶意评价 |

**管理员操作流程：**
```
1. 点击页面右上角"管理"按钮
2. 输入管理员密码
3. 验证通过，进入管理界面
4. 编辑今日状态或历史记录
5. 保存，数据写入 KV
6. 退出登录（可选）
```

## 🚀 部署指南

### 步骤 1：初始化项目

```bash
# 创建项目目录
mkdir shaobing
cd shaobing

# 创建目录结构
mkdir -p public/images public/css public/js functions/api
```

### 步骤 2：配置 wrangler.toml

```toml
name = "shaobing"
compatibility_date = "2026-03-18"

[site]
bucket = "./public"

[[kv_namespaces]]
binding = "KV"
id = "your_kv_namespace_id"

[vars]
ADMIN_PASSWORD_HASH = "$2a$10$..."  #  bcrypt 哈希后的密码
MAX_REVIEWS = 500
MAX_HISTORY_DAYS = 90
```

### 步骤 3：初始化 KV 数据

```bash
# 创建 KV 命名空间
wrangler kv:namespace create "KV"

# 将返回的 id 填入 wrangler.toml

# 初始化管理员密码（使用 bcrypt 哈希）
wrangler kv:key put --binding=KV "admin:password" "$2a$10$..."

# 初始化今日状态
wrangler kv:key put --binding=KV "status:2026-03-18" '{"date":"2026-03-18","isOpen":false,"status":"休息","location":"-","time":"-","note":"尚未出摊","updatedAt":"2026-03-18T00:00:00Z"}'

# 初始化历史记录
wrangler kv:key put --binding=KV "history" "[]"

# 初始化评价
wrangler kv:key put --binding=KV "reviews" "[]"
```

### 步骤 4：部署命令

```bash
# 本地开发
wrangler pages dev ./public --kv=KV

# 部署到 Cloudflare Pages
wrangler pages deploy ./public
```

## 📅 维护手册

### 日常运营流程（管理员）

```
06:30  登录后台，编辑"今日状态"
       - 设置状态：出摊中
       - 填写位置：xx 小区门口
       - 填写时间：07:00-09:00
       - 添加备注：今天多带了韭菜盒子

09:30  收摊后更新状态
       - 修改状态：已收摊
       - 补充备注：今天带了 50 个烧饼，全部售罄

23:00  自动归档（Worker 定时任务）
       - 今日状态自动存入历史记录
       - 清理 90 天前的记录
```

### 评价管理

| 操作 | 说明 |
|------|------|
| 查看评价 | 登录后台 → 评价管理 → 查看所有评价详情 |
| 删除恶意评价 | 点击评价右侧 [删除] 按钮 |
| 导出评价 | 点击 [导出] 按钮，下载 JSON 格式评价数据 |

### 数据清理

| 数据类型 | 清理规则 |
|---------|---------|
| 历史记录 | 保留最近 90 天，自动清理 |
| 评价数据 | 保留最近 500 条，自动清理 |
| 登录会话 | 24 小时过期，需重新登录 |

## 🔒 安全策略

| 风险 | 防护手段 |
|------|---------|
| 未授权访问 | 管理员密码验证（bcrypt 哈希存储） |
| 刷评 | IP 速率限制（每小时 1 次/IP） |
| 恶意提交 | 敏感词过滤 + 内容长度限制 |
| 数据泄露 | KV 私有存储，不暴露敏感信息 |
| CORS 攻击 | 同域部署，严格 CORS 策略 |

## 🆘 常见问题

### Q: 如何修改管理员密码？
A: 在 Cloudflare Dashboard → KV → 浏览密钥 → 修改 `admin:password` 的值（需使用 bcrypt 哈希）。

### Q: 图片如何更换？
A: 将新图片放入 `public/images/` 目录，修改 `index.html` 中的图片路径。

### Q: 收款码如何更换？
A: 将微信/支付宝收款码图片命名为 `wechat-qr.png` 和 `alipay-qr.png`，放入 `public/images/` 目录。

### Q: 如何查看历史评价？
A: 登录管理后台 → 点击"评价管理"标签 → 查看所有评价详情。
